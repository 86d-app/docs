## How 86d works

86d is organized as a monorepo of composable modules. Each module is an isolated npm package that contributes API endpoints, admin UI, and storefront components. You enable only the modules your store needs — in `config.json` — and they are automatically wired into your storefront and admin dashboard.

<Steps>
  <Step title="Install and initialize">
    Install the CLI, run `86d init` to configure your environment, database, and seed data.
  </Step>
  <Step title="Choose your modules">
    Add modules to `templates/brisa/config.json`. Every `@86d-app/*` module on npm is available.
  </Step>
  <Step title="Run codegen">
    Run `86d generate` to regenerate module imports and the API router. Components are now available in your MDX templates.
  </Step>
  <Step title="Customize and deploy">
    Edit your MDX templates to control layout and content. Deploy to Railway, Vercel, or any Docker host.
  </Step>
</Steps>
