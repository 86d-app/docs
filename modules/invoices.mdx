---
title: "Invoices"
description: "Invoice lifecycle for 86d: draft, sent, paid, partial, overdue. Issue invoices to B2B customers with payment terms and credit notes."
---

The invoices module manages an invoice lifecycle separate from order receipts. Invoices are intended for B2B contexts: extended payment terms (Net 30, Net 60), credit notes, partial payments, and an aging report.

**Source:** [`modules/invoices`](https://github.com/86d-app/86d/tree/main/modules/invoices) · **npm:** [`@86d-app/invoices`](https://www.npmjs.com/package/@86d-app/invoices)

## Installation

```sh
86d module add invoices
```

## Store endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/orders/me/:id/invoice` | Get invoice for an order |
| `GET` | `/orders/me/:id/invoice/pdf` | Download as PDF |

## Admin endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/admin/invoices` | List with status and aging filters |
| `POST` | `/admin/invoices/:id/send` | Email the invoice |
| `POST` | `/admin/invoices/:id/credit-note` | Issue a credit note |
