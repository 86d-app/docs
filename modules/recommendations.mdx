---
title: "Recommendations"
description: "Personalized product recommendations powered by OpenAI, Gemini, or OpenRouter. Falls back to rule-based suggestions when no AI key is configured."
---

The recommendations module surfaces products tailored to each shopper. With an LLM key configured (`OPENAI_API_KEY`, `GEMINI_API_KEY`, or `OPENROUTER_API_KEY`), recommendations use embeddings and the shopper's recently-viewed and order history. Without a key, the module falls back to a rule-based recommender (shared category, shared tags, co-purchase frequency).

**Source:** [`modules/recommendations`](https://github.com/86d-app/86d/tree/main/modules/recommendations) · **npm:** [`@86d-app/recommendations`](https://www.npmjs.com/package/@86d-app/recommendations)

## Installation

```sh
86d module add recommendations
```

## Configuration

```ts
recommendations({
  provider: "openai",       // "openai" | "gemini" | "openrouter" | "rule-based"
  model: "gpt-4o-mini",
  cacheMinutes: "60",
});
```

## Store endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/recommendations/for-product/:id` | "Customers also liked" for a product |
| `GET` | `/recommendations/for-customer` | Personalized for the authenticated shopper |
| `GET` | `/recommendations/trending` | Trending products globally |

## Components

- `<ProductRecommendations productId={id} />`: drop on product detail pages.
- `<PersonalizedShelf />`: home or account-page shelf.
- `<TrendingProducts />`: editorial shelf.
