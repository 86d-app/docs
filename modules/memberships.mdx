---
title: "Memberships"
description: "Paid membership plans in 86d. Gate products, content, and discounts behind a recurring fee."
---

The memberships module sells access plans rather than physical products. Members pay a recurring fee and unlock gated catalog items, member-only pricing, and exclusive content. The recurring billing piece runs through `@86d-app/payments` and `@86d-app/subscriptions`.

**Source:** [`modules/memberships`](https://github.com/86d-app/86d/tree/main/modules/memberships) ┬Ę **npm:** [`@86d-app/memberships`](https://www.npmjs.com/package/@86d-app/memberships)

## Installation

```sh
86d module add memberships
```

## Store endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/memberships/plans` | Available plans |
| `POST` | `/memberships/subscribe` | Subscribe to a plan |
| `GET` | `/memberships/me` | Active membership and benefits |

## Admin endpoints

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/admin/memberships/plans` | Create a plan |
| `GET` | `/admin/memberships/members` | List members |
| `POST` | `/admin/memberships/:id/cancel` | Cancel a membership |

## Components

- `<MembershipPlans />`, `<MembershipBadge />`.
