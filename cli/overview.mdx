The `86d` CLI is the primary tool for working with your store locally. You use it to start the development server, configure a new store, create and manage modules and templates, and keep generated code in sync after you make changes to `config.json`.

## Installation

Install the CLI globally with npm:

```sh
npm install -g 86d
```

If you are working inside the 86d monorepo, you can run the CLI without a global install:

```sh
bun run 86d <command>
```

## What the CLI does

The CLI covers four main areas:

- **Scaffolding** — generate a new module or copy a template as a starting point for a custom design
- **Development server** — start the Next.js storefront on your local machine
- **Code generation** — regenerate module imports, the API router, and component documentation after you edit `config.json`
- **Module management** — enable or disable modules in your active template without manually editing config files

## Command reference

| Command | Description |  |
| --- | --- | --- |
| `86d dev` | Start the store development server |  |
| \`86d init \[--yes | -y\]\` | Configure a local store |
| `86d status` | Show project health and configuration |  |
| `86d module create <name>` | Scaffold a new module |  |
| `86d module list` | List all modules |  |
| `86d module info <name>` | Show module details |  |
| `86d module enable <name>` | Enable a module |  |
| `86d module disable <name>` | Disable a module |  |
| `86d template create <name>` | Scaffold a new template |  |
| `86d template list` | List all templates |  |
| `86d template activate <name>` | Switch to a template |  |
| `86d generate` | Run all code generation |  |
| `86d generate modules` | Generate module imports |  |
| `86d generate components` | Generate component docs |  |

## Typical development workflow

This sequence covers the common setup-and-develop loop:

```sh
# 1. Configure the store — sets up .env, installs deps, runs migrations, seeds demo data
86d init

# 2. Check that everything is healthy before you start
86d status

# 3. Start the development server
86d dev

# 4. Scaffold a new module when you need custom functionality
86d module create loyalty-points

# 5. Enable it in your active template
86d module enable loyalty-points

# 6. Regenerate imports and the API router so the module is wired in
86d generate
```
